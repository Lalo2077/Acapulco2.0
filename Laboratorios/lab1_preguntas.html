Laboratorio 1: Introducción a las aplicaciones web, HTML5 y ciclo de vida de los sistemas de información

  • ¿Cuál es la diferencia entre Internet y World Wide Web?
    • Internet es la red de dispositivos en el mundo conectados entre sí y la world wide web son las páginas a las que uno puede acceder.

  • ¿Cuáles son las partes de un URL?
    • Protocolo / Subdominio / Dominio / TLD (nivel de dominio superior) / Ruta / Parámetro / Etiqueta

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • - HTTP: Realizar peticiones de datos o información.
    - GET: Petición para recuperar un dato de un recurso en específico.
    - HEAD: Pide una respuesta idéntica a la petición GET pero sin el cuerpo de la respuesta.
    - POST: Se usar para enviar información a un recurso en específico.
    - PUT: Hace una modificación de todo el recurso.
    - PATCH: Es utilizado para aplicar modificaciones parciales a un recurso.
    - DELETE: Borra un recurso.

  • ¿Qué metodo HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    • Los más usados son GET y POST
    GET se usa para recuperar recursos y POST recupera recursos basados en datos proporcionados en el cuerpo de la petición.

  • ¿Qué método HTTP se utilizar cuando a través de un navegador web se accede a una página a través de un URL?
    • El método GET.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    • No ocurre ningún error, significa que la solicitud ha tenido éxito.
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    • No es responsabilidad del desarrollador, ya que un error 404 es de parte del usuario.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué??
    • Si, ya que este tipo de error es proveniente de la parte interna del sitio web y el autor o desarrollador tiene que corregir este error para que no se vuelva a dar.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elemtnso de HTML4 que en HTML5 estén desaprobados.
    • Esto significa que su uso ya no es recomendado en HTML5
    Ejemplos:
    applet reemplazado por object

  • ¿Cuáles son las diferencias principales entre HTML4 y HTML5?
    • HTML5 se respalda de CSS y JavaScript mientras que la versión de HTLM4 lo hacía todo.
    HTML5 requiere doctype para su uso correcto.

  • ¿Qué componentes de estructura y estilo tiene una tabla?
    • Tabla (crear tabla), tr(filas), th(celdas de cabezera), td(celda de datos), tbody(body de la tabla), theader (header de la tabla), tfoot (footer de la tabla).

  • ¿Cuáles son los principales controles de una forma HTML5?
    • input type = "number" = campo de entrada para números, que le permite aumentar o disminuir su valor.
    input type = "range" = crea un control deslizante que le permite elegir entre un rango de valores.
    input type = "date" = crea un selector de fecha totalmente funcional.
    input type = "color" = selector de color.
    input type = "search" = campos de texto diseñado para que el usuario use para hacer busquedas.
